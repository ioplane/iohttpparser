# Documentation

Stable documentation tree for `iohttpparser`.

## Structure

| Path | Role |
|---|---|
| [`en/`](en/README.md) | Authoritative English documentation |
| [`ru/`](ru/README.md) | Russian translation layer |
| [`plans/`](plans/README.md) | Roadmaps, execution plans, and reports |
| [`rfc/`](rfc/README.md) | Local RFC mirror |
| `tmp/` | Scratch and non-authoritative working material |

## Stable Entry Points

| Document | Purpose |
|---|---|
| [`en/README.md`](en/README.md) | English documentation index |
| [`ru/README.md`](ru/README.md) | Russian documentation index |
| [`plans/README.md`](plans/README.md) | Plans index |
| [`plans/ROADMAP.md`](plans/ROADMAP.md) | Current roadmap |
| [`plans/BACKLOG.md`](plans/BACKLOG.md) | Deferred work |

## Rules

- `docs/en/*` is authoritative.
- `docs/ru/*` follows the English structure and scope.
- Numbered documents in `docs/en/` and `docs/ru/` must match by topic.
- `docs/tmp/*` is excluded from the stable documentation surface.
